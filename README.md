# Food-Menu-Management-System
                                                      **Project Intent**
The core purpose of this project is to sharpen my programming skills by taking concepts out of tutorials and applying them to a real, functional codebase. This is Version 1.0.0, shipped as a working foundation that I intend to structurally upgrade in the future.

                                                      **What it Does**
Add Dishes: Takes a dish name and price from the interface and saves it directly to a Menu.txt file.

View Menu: Reads the data file and displays the current restaurant menu back to the user inside a clean window.

                                                 ** How it Was Built**
This project is a direct combination of my learning and strategic AI engineering:

My Execution: I designed the modular multi-file architecture (Main.py, GUI.py, Inventory_Management.py) to keep the interface separated from the backend logic, wrote the core data handling logic, and implemented the standard layout.

AI Assistance: I collaborated with an AI assistant to debug specific syntax blockers (such as class method indentation and theme naming rules), transition terminal input() statements into smooth GUI event handlers, and ensure safe runtime text box manipulation.
